Scope: FlashVSR DiT PTQ only. Wan VAE/decoder remain unquantized.

## Implemented

- Cherry-picked prerequisite A4 activation infrastructure from `feat/lsgquant-a4-activation-q`:
  - `feat(ptq): add A4 activation fakequant mode`
  - `fix(ptq): infer mixed fakequant modes at load`
- Added PR9 A4W4 fallback policy builder:
  - low VOLTS sensitivity → `a4w4` + DRAQ + LSGQuant rank-16
  - mid VOLTS sensitivity → `a4w4` + DRAQ + LSGQuant rank-32 + light adaptation marker
  - high VOLTS sensitivity → `a8w8` + DRAQ + LSGQuant rank-32 fallback
  - optional top-k catastrophic layers → `fp16_skip`
- Extended `scripts/ptq/lsgquant_policy.py` with `--policy_type a4w4_fallback` and `--fp16_topk`.
- Extended `scripts/ptq/run_lsgquant_eval_set.py` for PR9:
  - accepts `--mode a4w4`, `--rank`, `--qao_rounds`, `--rotation`
  - writes manifest schema `flashvsr.lsgquant.a4w4_eval_manifest.v1`
  - calls `scripts/ptq/lsgquant_convert.py` instead of fakequant_convert for A4W4 QAO conversion
  - records fallback counts, QAO config, compression/deployment boundary, and pending quality-delta fields
- Extended QAO conversion to accept `a4w4`, honor per-layer policy rank, and allow `a8w8` high-sensitivity fallback inside an A4W4 default policy.
- Added `docs/quantization/a4w4_results.md` documenting commands, verification, and the no-speed/no-quality-claim boundary.
