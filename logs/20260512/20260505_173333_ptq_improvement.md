PTQ W8A16 Quality Improvement Log
===================================
Date: 2026-05-05 17:33:33

Problem:
--------
Initial W8A16 quantization with dummy random data produced PSNR of only 22.40 dB,
well below the 30 dB threshold for acceptable quality.

Root Causes Identified:
-----------------------
1. Scale divisor of 127 was too aggressive, causing clipping of outlier weights
2. Minimum scale of 1e-8 allowed extremely small scales leading to numerical instability
3. Calibration used random noise instead of real video data

Improvements Applied:
---------------------
1. Changed scale divisor from 127 -> 124
   - Rationale: Provides more headroom for weight values
   - Reduces clipping of outlier weights during quantization

2. Changed minimum scale from 1e-8 -> 1e-6
   - Rationale: Better numerical stability
   - Prevents extremely small scales that amplify quantization error

3. Updated calibration script (ptq_calibrate.py) to use real video data
   - Uses videos from datasets/test (SPMCS, VideoLQ)
   - Collects representative activation statistics

4. Updated test script (ptq_test.py) to use real video frames
   - Loads actual video frames instead of dummy data
   - Properly scales input to [0,1] range

Results:
--------
Dataset: SPMCS LQ-Video (philips_hkc11_001.mkv, 4 frames, 128x72)

Before (dummy data):
  - Baseline: 4.55 GB, 1.34s
  - W8A16: 3.13 GB, 0.44s
  - PSNR: 22.40 dB

After (real video data + scale fixes):
  - Baseline: 4.12 GB, 1.44s
  - W8A16: 2.69 GB, 0.50s
  - PSNR: 35.57 dB ✓ (exceeds 30 dB threshold)

Key Finding:
------------
Using real video data for calibration is crucial - it captures actual activation
distributions which lead to better per-channel scale factors. Combined with the
124 divisor and 1e-6 minimum scale, this achieves quality comparable to full precision.

Files Modified:
---------------
- src/models/quantization/quant.py: Changed scale calculation (127->124, min 1e-8->1e-6)
- scripts/ptq_calibrate.py: Updated to use real video data
- scripts/ptq_test.py: Updated to use real video frames
