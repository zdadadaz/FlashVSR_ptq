# W8A8 量化技術說明：BF16 vs. INT8MM

本文件紀錄了關於 W8A8 量化中，使用 BF16 與 INT8MM 算子的差異，以及量化對權重與計算過程的影響。

## 1. W8A8 與 BF16 的關係

在 W8A8 的設定下使用 `bf16`，通常代表以下情況：

*   **反量化計算 (Dequantization-on-the-fly):**
    *   **存儲：** 權重 (Weight) 和激活值 (Activation) 在記憶體中以 8-bit (int8) 形式存在，節省顯存。
    *   **計算：** 在矩陣乘法前，數據被立即轉回 `bf16`。
    *   **影響：** 雖然節省了顯存（VRAM），但因為運算核心仍是浮點數，**推理速度通常不會加速**，甚至可能因為轉換開銷而略微變慢。

## 2. BF16 vs. INT8MM 的核心差異

| 特性 | W8A8 (使用 BF16 計算) | W8A8 (使用 INT8MM 計算) |
| :--- | :--- | :--- |
| **權重精度** | 8-bit (存儲) | 8-bit (存儲) |
| **激活值精度** | 8-bit (存儲/轉換) | 8-bit (參與計算) |
| **核心運算** | **BF16 浮點數運算** | **INT8 整數矩陣乘法** |
| **硬體利用** | 使用 CUDA Core 或 BF16 Tensor Core | 使用 **INT8 Tensor Core** |
| **加速效果** | 極小（主要節省顯存） | **顯著加速 (理論上 > 2x)** |
| **精度控制** | 較容易 | 較難 (需精確處理 Scale 與溢出) |

## 3. 量化對象：權重 vs. 計算

*   **W8A16 (僅權重量化):**
    *   權重為 int8，激活值為 16-bit。
    *   計算前權重轉回 16-bit，執行 **16-bit 運算**。
    *   主要目的：減小模型體積、節省帶寬。
*   **W8A8 + int8mm (全量化計算):**
    *   權重與激活值均被量化為 int8。
    *   硬體直接執行 **int8 * int8 = int32** 的整數運算。
    *   主要目的：極致的**推理速度**與最低顯存佔用。

## 結論

1.  **W8A8 with bf16**：屬於「省顯存、不省時間」的偽量化或模擬量化。
2.  **W8A8 with int8mm**：是真正的量化推理，利用硬體整數加速單元追求極速。
3.  **計算量化**：只有當激活值也參與量化並使用 `int8mm` 類型的整數算子時，運算過程才真正被量化。
