# 20260513_132751 FFmpeg Sync Fix

## Objective
Fix synchronization issues when combining original and upscaled videos.

## Diagnosis
- `-r 30` was forcing a frame rate that might not match the source, leading to drift.
- `-vsync 2` allowed jitter between the two input streams.
- Missing PTS (Presentation Timestamp) alignment meant the videos might not start at the same "time zero".

## Changes
- Modified `ffmpeg_comb.sh`.
- Added `setpts=PTS-STARTPTS` to both streams to align their starting points.
- Removed `-r 30` and `-vsync 2` to preserve original timing.
- Added `format=yuv420p` for standard compatibility.

## Execution Result
- `ffmpeg_comb.sh` updated for frame-accurate side-by-side comparison.
